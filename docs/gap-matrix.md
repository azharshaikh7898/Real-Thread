# Real-Time Threat Monitoring Platform Gap Matrix

Date: 2026-04-16

This matrix compares the target SOC-style project scope with the currently implemented repository capabilities.

## Legend

- Priority P0: Required for project objective and core deliverables
- Priority P1: Important for detection quality and analyst efficiency
- Priority P2: Optional/advanced enhancements
- Status: Implemented, Partial, Missing

## Gap Matrix

| Requirement Module | Current Status in Repo | Priority | Exact Implementation Tasks | Acceptance Criteria |
|---|---|---|---|---|
| 1. Data Source Onboarding (endpoints, network, web, identity, cloud) | Partial. No native source-specific onboarders, collectors, or parsers per telemetry type. | P0 | Add source adapters/parsers for Windows auth events, Linux auth logs, Nginx access logs, firewall syslog, identity logs (at least one identity source). Add ingestion health metrics by source and parse-failure tracking. Add timestamp drift/time-sync validation checks at ingest. | At least 5 distinct source types can be ingested and parsed into normalized records. ## Current Strengths to Preserve

- End-to-end ingest -> detect -> alert -> dashboard flow is already implemented.
- Authentication and role-aware endpoint protection are present.
- Real-time updates through WebSocket are implemented.
- Baseline unit tests exist for detector behavior.

## Suggested Execution Order

1. P0 foundational data quality: onboarding validation + normalization/enrichment schema.
2. P0 detection expansion: complete 8-12 MITRE-mapped rules and test harness.
3. P0 analyst workflow: case management, triage, investigation timelines.
4. P1 visibility and quality: dashboard expansion + tuning lifecycle and metrics.
5. P1/P2 operational maturity: playbooks, optional automation, final report packaging.
